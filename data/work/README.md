## 工作结构

工作日上班时间高度稳定，通常在 08:48–08:59。周一至周四常见下班时间约 20:05，周五通常较早，约 18:00–19:20。电瓶车通勤十几分钟，因此“到家晚”不是唯一问题，真正影响晚间学习的是下班前任务集中、精神未收口以及回家后恢复时间失控。

## 核心判断

1. 周一至周四不能按普通 9–18 点工作日设计，晚间真实容量通常只有 45–100 分钟。
2. 周五、较早下班日和无加班周末才是高质量学习块的主要来源。
3. 白天摸鱼如果导致任务压到下班前，会把名义上的晚间时间变成被迫恢复时间。
4. 加班已经从 5 月的高频状态下降，但仍需要按月预算，不能把每个可加班日都当作默认收入机会。

## 下班前收口协议

每天 17:00 左右做一次工作切片：

```markdown
今天必须完成：
今天做到什么程度即可交付：
可以明天继续：
今天不再新开：
预计下班：
```

下班前 60–90 分钟停止无目的切换、刷网页和打开新任务，只做交付与收尾。目标不是全天高压，而是把工作疲劳留在公司。

## 按工作日类型调度

| 类型 | 条件 | 晚间安排 |
|---|---|---|
| 长工时日 | 20 点左右下班且任务拖尾 | 45–75 分钟，旧题/选择/复盘 |
| 正常长工时日 | 20 点左右下班但工作已收口 | 75–100 分钟，唯一主任务 |
| 早下班日 | 18–19 点下班 | 150–180 分钟，数学或 408 主攻 |
| 周末长加班 | 21 点后下班 | 只保连续性，优先睡眠 |
| 无加班周末 | 有完整半天 | 2.5–4 小时高质量块，分两段完成 |

## 加班预算

加班需要由收入缺口决定，而不是由“还能多挣一点”决定。每月先写：

```markdown
本月基本收入目标：
必要加班小时：
已完成加班小时：
剩余可接受加班：
超过预算后的代价：睡眠 / 周末高质量块 / 身体恢复
```

只有先把加班变成预算，工作和备考才不再无限争夺同一段时间。
