# 时间管理项目研究与 Horizon 取舍

目标不是复刻一个成熟效率软件，而是从成熟项目中提取能解决当前问题的机制：时间失真、任务超载、拖延启动、项目替代学习、缺少反馈闭环。

## 研究对象

### Super Productivity

核心能力：任务管理、子任务、项目、标签、时间盒、实际耗时、时间表、休息提醒、番茄钟、抗拖延提示、个人指标，并可接入 Jira、GitHub 等任务来源。

对 Horizon 的启发：

1. 任务必须同时有预计时间和实际时间。
2. 任务不是越多越好，而是先进入固定时间盒。
3. 休息提醒属于生产系统，不是附属功能。
4. 个人指标用于修正工作方式，不用于制造数据负担。

不采用：复杂看板、多项目自动导入、大量标签、完整番茄钟系统。当前阶段这些功能容易让 Horizon 本身变成主项目。

### ActivityWatch

核心能力：在本地自动记录活跃应用、窗口标题、浏览器标签、键鼠活动和离开电脑时间，用真实数据回答时间去了哪里。

对 Horizon 的启发：

1. 不能依赖“我今天好像学了很久”的回忆。
2. 必须区分考试训练、项目建设、搜索讨论和纯消耗。
3. 先观察，再干预；没有事实就不要重构计划。

可选后续：手工流程稳定两周后，再接入 ActivityWatch。当前只记录计划分钟、实际有效分钟和主要漏损，不先做自动化。

### Taskwarrior

核心能力：可查询的命令行任务系统，支持项目、依赖、筛选、钩子和扩展。

对 Horizon 的启发：

1. 任务要有状态，而不是只有一张清单。
2. 任务要有前置条件、下一动作和阻塞原因。
3. 系统应能从大量候选任务中筛出当前唯一动作。

不采用：复杂命令语法和完整任务数据库。Horizon 当前只需要稳定的 Markdown 结构。

## 提炼出的控制模型

```text
真实状态输入
→ 估计今日容量
→ 从候选任务中筛出唯一主任务
→ 设定完成线与时间上限
→ 执行并记录实际耗时
→ 标记一个主要漏损
→ 周复盘时只修最高频漏洞
```

这个模型比“列出今天所有要做的事”更适合当前状态。它承认精力有限，并要求系统主动舍弃。

## 四个核心机制

### 1. Timebox：以时间上限约束任务

每个主任务必须写：

```markdown
任务：
预计：
完成线：
时间上限：
```

时间到后停止，不继续扩展知识树、搜索资料、改项目结构。未完成部分转成下一动作，而不是当晚无限延长。

### 2. Actual：记录真实有效时间

只记录有效训练，不记录“坐在电脑前”的时长。

```markdown
预计：90 分钟
实际有效：62 分钟
差值原因：启动晚 / 手机 / AI 扩展 / 工作拖尾
```

连续一周后，用中位实际时长修正下一周计划。计划能力本身需要训练。

### 3. WIP Limit：限制同时进行的任务

每日在制品上限为：

- 1 个主任务
- 0–1 个维持任务
- 1 个明确不做清单

四科并行不是每天四科都做，而是在周尺度分配主次。工作日晚间不得同时展开数学、408、英语精读和系统开发。

### 4. Leak Review：只修一个最大漏洞

漏损代码：

```text
S：睡眠不足
W：工作拖尾
R：恢复无边界
P：项目替代学习
D：手机 / 网页分心
T：任务过大
U：任务定义不清
```

每天最多标一个。周末统计最高频项，下周只针对它增加一条规则。避免同时改睡眠、运动、饮食、工具和四科学习，导致系统再次超载。

## 对 Horizon 的产品约束

1. 每日记录不超过 5 分钟。
2. 周复盘不超过 15 分钟。
3. 每周结构维护不超过 60 分钟。
4. 新功能必须解决连续两周出现的真实问题。
5. 没有考试训练产出时，禁止新增功能。
6. 自动化只能建立在已经手工跑通的流程上。
7. 指标必须能改变下一步决策，否则删除。

## 当前最小指标集

| 指标 | 用途 |
|---|---|
| 平均睡眠 | 决定容量能否上调 |
| 工作拖尾天数 | 判断晚间恢复成本来源 |
| 有效考试训练分钟 | 判断主线是否真正发生 |
| 高质量块数量 | 判断数学、408是否获得整块时间 |
| 项目建设分钟 | 防止构建替代训练 |
| 最大漏损代码 | 决定下一周唯一优化点 |

不跟踪：commit 数、笔记字数、打开资料数量、与 AI 对话长度。这些数据容易增长，但不能直接证明考试能力增长。

## 后续可能接入的能力

只有在手工运行稳定后，才考虑：

- 从 Apple 健康数据自动读取睡眠
- 从 ActivityWatch汇总软件使用时间
- 从考勤记录识别工作日类型
- 从学习仓库提取真实训练产物
- 自动生成当日容量和候选任务

自动化的终点不是生成更长的日报，而是把每天的选择压缩成一句：

> 今天在现有容量下，哪一个动作最值得完成。
